## 🌟 愿景与理念

### 我们的使命

> **"消除人与软件之间的最后一道鸿沟。"**

在数字化时代，软件已成为我们工作和生活的核心工具。然而，学习新软件的成本却越来越高：
- 📚 企业员工需要花费数周时间学习内部系统
- 💼 新入职员工需要大量培训才能上手复杂软件
- 🎓 学生和创作者需要反复查阅教程才能掌握专业工具
- 🏢 企业每年投入巨额成本进行软件培训

**我们相信，未来的人机交互不应该让用户去"学习"软件，而是让软件"理解"用户。**

### 产品理念

AI Navigation System 不是另一个教程网站，不是另一个聊天机器人，而是一个**真正"看得见"你屏幕的 AI 助手**。

#### 🎯 核心创新点

1. **实时屏幕感知**
   - 不是基于预设模板，而是实时读取你的屏幕
   - 理解你当前的操作上下文
   - 动态适应不同的软件界面

2. **知识库驱动的智能引导**
   - 企业可以上传自己的操作手册和 SOP
   - AI 自动解析并生成可执行的引导路径
   - 支持私有化部署，保障企业数据安全

3. **零学习成本的交互方式**
   - 不需要记忆快捷键
   - 不需要查阅文档
   - 只需要跟随屏幕上的视觉引导

4. **跨软件通用架构**
   - 一套系统适配所有软件
   - 从设计工具到开发 IDE，从 CRM 到 ERP
   - 通过知识库扩展，支持任意软件

### 为什么现在？

- 🤖 **AI 技术成熟**：GPT-4V、Claude Vision 等视觉大模型已具备理解 UI 的能力
- 🖥️ **桌面技术成熟**：Electron、Tauri 等技术让跨平台桌面应用开发变得简单
- 📚 **RAG 技术成熟**：向量数据库和检索增强生成让知识库应用成为可能
- 🎨 **设计系统成熟**：苹果级的设计语言和交互范式已深入人心

**这是将 AI 真正融入人机交互的最佳时机。**

## 💼 工作流程

```
┌──────────────────────────────────────────────────────────────────┐
│  ① 上传知识库                                                      │
│     企业/用户上传软件说明书、操作手册、内部文档、SOP                 │
│     支持 .txt、.md、.pdf 等多种格式                                  │
│                              ↓                                     │
│  ② AI 智能解析                                                     │
│     自动提取操作步骤、UI 元素映射、异常处理逻辑                       │
│     生成向量化知识图谱，支持语义检索                                  │
│                              ↓                                     │
│  ③ 用户提出需求                                                     │
│     自然语言描述："帮我在 CRM 系统中创建一个客户档案"                  │
│     或选择预设任务："财务月结流程"、"构建销售流程"                     │
│                              ↓                                     │
│  ④ AI 实时规划                                                     │
│     基于知识库生成详细的操作步骤（5-30步，根据复杂度）                │
│     支持企业级复杂流程、多模块集成、错误处理                          │
│                              ↓                                     │
│  ⑤ 动态视觉引导                                                     │
│     屏幕上叠加高亮框 + 箭头，一步步指引操作                           │
│     实时感知屏幕状态，自动进入下一步                                  │
│                              ↓                                     │
│  ⑥ 任务完成 / 异常修正                                              │
│     自动检测错误操作，即时提供修正引导                                 │
│     支持回退、重试、跳过等操作                                       │
└──────────────────────────────────────────────────────────────────┘
```

## 🚀 核心能力

### 1. 企业知识库上传系统 (Enterprise Knowledge Upload)

**为什么重要？**
每个企业都有自己的内部系统、操作流程和业务逻辑。通用 AI 无法理解这些特定场景，而我们的系统可以让企业上传自己的知识库，AI 立即理解并生成专属引导。

**核心功能：**
- 📚 **文档智能解析**：自动解析 PDF、Word、Markdown、Confluence 等格式
- 🔄 **自动知识图谱生成**：将非结构化文档转化为可执行的操作逻辑树
- 🧠 **向量化存储**：使用语义检索快速定位相关操作指南
- 🏢 **私有化部署支持**：企业可部署专属知识库，保障数据安全
- 🔍 **智能步骤提取**：自动识别文档中的操作步骤和 UI 元素

### 2. 屏幕感知引擎 (Visual Perception Engine)

**技术实现：**
- 基于 Electron `desktopCapturer` 实现真实的全屏捕获
- 实时帧率监控（可达 2+ FPS 持续刷新）
- 支持多显示器环境

**未来规划：**
- 🔮 接入视觉大模型（GPT-4V / Claude Vision）进行 UI 元素识别
- 🎯 自动识别按钮、输入框、菜单等 UI 元素及其坐标
- 🔄 实时检测界面变化，动态调整引导路径

### 3. 任务规划器 (Task Planner)

**核心能力：**
- 🧩 **智能任务拆解**：将模糊的自然语言（如"帮我做一个登录页"）拆解为结构化步骤
- 📖 **基于知识库的 RAG**：使用检索增强生成，快速定位文档中的操作指南
- 🎯 **动态路径生成**：根据任务复杂度生成 5-30 步详细操作指南
- 🔀 **条件分支支持**：支持"如果出现 X 情况，则执行 Y"的复杂逻辑
- ⚠️ **错误处理机制**：包含回退、修正、验证等步骤
- 🏢 **企业级流程**：支持多角色协作、审批流程、数据同步等复杂场景

**复杂任务示例：**
- 企业级 ERP 系统配置（14 步）
- 多模块集成配置（14 步）
- 企业级 CRM 系统搭建（16 步）
- 财务月结流程（6 步）

### 4. 沉浸式交互层 (Immersive UX Layer)

**设计理念：**
遵循苹果级设计语言，追求极简、优雅、不打扰用户的交互体验。

**核心功能：**
- 🎯 **非侵入式高亮框**：精准标记目标 UI 元素，使用蓝色高亮和脉冲动画
- ➡️ **动态箭头指引**：实时引导用户视线，从侧边栏指向目标元素
- 💬 **悬浮指令面板**：苹果级设计，可随时与 AI 对话
- 📱 **响应式布局**：适配不同屏幕尺寸和分辨率
- 🎨 **极简美学**：蓝白配色，无渐变，符合现代设计趋势

### 5. 跨软件通用架构

**技术优势：**
- 🔌 **插件化设计**：通过知识库扩展支持任意软件
- 🎨 **界面适配**：自动适配不同软件的界面风格
- 🔄 **状态同步**：实时感知软件状态变化
- 📦 **模块化架构**：核心引擎与具体软件解耦

## 📦 快速开始

### 环境要求

- **Node.js** 18+ 
- **npm** 9+ 或 **yarn** / **pnpm**
- **Git** 用于克隆仓库

### 安装与运行

```bash
# 1. 克隆仓库
git clone https://github.com/mm1025048717-hash/ai-navigation-system.git
cd ai-navigation-system

# 2. 安装依赖
npm install

# 3. 配置环境变量
# 创建 .env.local 文件，添加你的 DeepSeek API Key
echo "DEEPSEEK_API_KEY=your-api-key-here" > .env.local

# 4. 启动开发模式（浏览器预览）
npm run dev

# 5. 启动桌面应用（真实屏幕读取）
npm run app
```

### 构建生产版本

```bash
# 构建 Next.js 应用
npm run build

# 构建 Electron 应用（需要 electron-builder）
npm run build:electron
```
