# 🚀 快速开始指南

## 问题排查

如果您遇到"还是不能用"的问题，请按以下步骤检查：

### 1. 检查开发服务器是否运行

```bash
# 确保开发服务器正在运行
npm run dev
```

应该看到：
```
✓ Ready in X seconds
○ Local: http://localhost:3000
```

### 2. 上传知识库文档

**步骤：**
1. 点击右侧侧边栏的 **"AI 对话"** 标签
2. 点击输入框左侧的 **上传按钮**（📤 图标）
3. 选择 `.txt` 或 `.md` 文件
4. 等待上传完成，会看到成功提示

**测试文件：**
项目根目录有 `test-doc.md` 文件，可以用它来测试上传功能。

### 3. 切换到智能引导

**步骤：**
1. 上传文档后，点击 **"智能引导"** 标签
2. 应该看到 "X 个文档已就绪"
3. 点击 **"继续"** 按钮
4. 选择软件（PyCharm / Reddit / Figma）
5. 选择任务类型（基础 / 复杂）
6. 等待AI生成步骤
7. 点击 **"开始智能引导"**

### 4. 常见问题

#### 问题：上传文件没有反应
**解决：**
- 检查文件格式是否为 `.txt` 或 `.md`
- 打开浏览器控制台（F12）查看错误信息
- 检查 `/api/upload` 接口是否正常

#### 问题：看不到引导箭头
**解决：**
- 确保在 Electron 模式下运行（`npm run app`）
- 浏览器模式下只能看到演示界面
- 检查 `Overlay` 组件是否正确渲染

#### 问题：步骤生成卡住
**解决：**
- 检查 API Key 是否配置（`.env.local`）
- 如果没有 API Key，系统会使用降级方案（预设步骤）
- 查看浏览器控制台的错误信息

### 5. 环境变量配置

创建 `.env.local` 文件：

```env
# DeepSeek API（推荐）
DEEPSEEK_API_KEY=sk-your-key-here

# 或 OpenAI API
OPENAI_API_KEY=sk-your-key-here
```

### 6. 完整启动流程

```bash
# 1. 安装依赖（如果还没安装）
npm install

# 2. 配置环境变量
# 编辑 .env.local，添加 API Key

# 3. 启动开发服务器
npm run dev

# 4. 在另一个终端启动 Electron（可选，用于真实屏幕捕获）
npm run app
```

### 7. 测试功能

1. **测试上传：**
   - 使用项目根目录的 `test-doc.md`
   - 上传后应该看到成功消息

2. **测试引导：**
   - 选择 Reddit + 复杂任务 + "积累 1000 粉丝"
   - 应该生成 6 个步骤

3. **测试对话：**
   - 在 AI 对话中输入问题
   - 应该收到基于知识库的回答

## 如果还是不行

请提供以下信息：
1. 浏览器控制台的错误信息（F12）
2. 终端中的错误日志
3. 您执行的具体操作步骤

这样我可以更准确地帮您解决问题！
